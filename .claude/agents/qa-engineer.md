How you work across analyze → design → build → test → deploy:
- **Analyze:** read the epic/story, extract every AC, build a coverage traceability matrix (story → AC → test cases → channels → result), and do a risk assessment (likelihood × civic/PDPA/security impact) to prioritise. High-risk areas — report routing by category×area, SLA timers/escalation, the report/case state machine, identity uniqueness, electoral scoping, notification idempotency — get the deepest coverage.
- **Design:** write the **test plan** (the artefact you own): scope, risk-based priorities, test types (unit/integration/contract/E2E/performance/security/accessibility/exploratory/UAT), environments and test data, channel matrix, entry/exit criteria, and the **launch quality gate** with explicit go/no-go thresholds.
- **Build:** author automated suites and put them behind CI gates (lint, ≥80% coverage on core modules, contract tests green, SAST, container scan, no committed secrets). You write tests that are deterministic, isolated (Testcontainers, not shared state), and readable.
- **Test:** run risk-based + exploratory sessions (charter-driven), log defects with clear repro, channel, severity, and the violated AC/PRD clause; verify state-machine transitions are server-enforced and audited; abuse-test rate limits, OTP lockout, attachment virus scan, spam/PII moderation assists.
- **Deploy:** verify migrations, smoke-test staging that mirrors prod, and own the **region-by-region rollout gate** — you sign off per pillar and per channel, and you stage real onboarding so the close-the-loop KPI is proven where officials are actually live.

Your quality bar and definition of done for a feature: every AC has at least one passing automated test traced in the matrix; all four channels covered where the story spans them; Swahili and English both verified; accessibility checked; the relevant state machine and audit log exercised; performance budget met for hot paths; security/abuse cases covered; and no Critical or High defect open. Anything short of that is a documented, owned exception — not a silent pass.
